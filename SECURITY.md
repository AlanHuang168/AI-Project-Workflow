# Security Policy

## Supported Versions

The current `0.x` release line receives security fixes when maintainers are able to provide them.

## Reporting Vulnerabilities

Please do not open public issues for security vulnerabilities.

Please use GitHub private vulnerability reporting when available.

## What to Include

- Affected version.
- Description of the issue.
- Reproduction steps or proof of concept when safe to share.
- Potential impact.
- Suggested mitigation, if known.

## Response

Maintainers will review reports as soon as possible. Do not publicly disclose the vulnerability until maintainers have had a reasonable opportunity to respond.

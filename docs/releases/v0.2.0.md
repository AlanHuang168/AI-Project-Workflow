# AI Project Workflow v0.2.0

## Highlights

- Configurable workflows through `.apw/project.yaml`.
- Project-level custom Skills under `.apw/skills/<stage>/SKILL.md`.
- Config-aware `init`, `status`, and `validate`.
- Real-environment compatibility reports for Cursor, Claude Code, Codex CLI, TRAE, and CodeBuddy.

## Configurable workflows

Projects without `.apw/project.yaml` continue using the default eight-stage workflow:

```text
init -> prd -> hld -> sdd -> impl -> review -> deploy -> retro
```

Projects may define custom stage order:

```yaml
protocolVersion: 1.0.0
stages:
  - init
  - discovery
  - impl
  - review
```

Custom project Skills live at `.apw/skills/<stage>/SKILL.md`.

## Installation

```bash
npx @dayahs/ai-project-workflow@0.2.0 init . --platform cursor
```

## Upgrade notes

Existing projects without `.apw/project.yaml` continue to use the default eight-stage workflow.

After upgrading, run:

```bash
apw install . --platform <platform>
apw validate .
```

Existing user files are preserved by default. Incoming generated changes are written as `.ai-sdd.new` unless `--force` is used.

## Known limitations

- Qoder real-environment verification is pending because the current test account has no remaining usage credits.
- The `.apw/project.yaml` parser supports only the documented lightweight subset, not full YAML.
- Workflow enforcement remains prompt-based and depends on each AI coding tool loading the generated context.
